# دليل إشعارات الدفع (Push) — تصل والتطبيق مغلق

> هذه الميزة تحتاج جزءاً خلفياً: **Firebase Cloud Messaging** + **دالة سحابية** تُرسل الإشعار عند إنشاء الفاتورة. وتتطلب **خطة Blaze (فوترة)** و**Firebase CLI**. الإشعار داخل التطبيق (والتطبيق مفتوح) يعمل بدون هذا.
>
> ⚠️ لا يمكن اختبار هذا إلا على مشروعك الفعلي. نفّذ الخطوات بالترتيب.

## كيف تعمل
المالك يسجّل فاتورة لزبون مسجّل → تُكتب في Firestore بحقل `customerId` → دالة سحابية تُطلَق تلقائياً → تقرأ رموز أجهزة هذا الزبون وترسل إشعاراً عبر FCM → يظهر على هاتفه حتى لو كان التطبيق مغلقاً.

---

## المرحلة ١ — تفعيل Cloud Messaging ومفتاح VAPID
1. Firebase Console → **Project settings (⚙️) → Cloud Messaging**.
2. تحت **Web Push certificates** → **Generate key pair**.
3. انسخ المفتاح (Key pair) والصقه في `index.html`:
   ```js
   const FCM = { enabled: true, vapidKey: 'المفتاح_هنا' };
   ```

## المرحلة ٢ — ملفات الموقع
ضع في **جذر** المستودع (HTTPS):
- `firebase-messaging-sw.js` (مرفق).
- `icon-192.png` (لأيقونة الإشعار).
ثم انشر `index.html` المحدّث.

## المرحلة ٣ — ترقية المشروع لخطة Blaze
الدوال السحابية تتطلب فوترة. Console → **Upgrade** → خطة **Blaze** (هناك حصة مجانية شهرية كبيرة تكفي الاستخدام البسيط).

## المرحلة ٤ — نشر الدالة السحابية
على حاسوبك:
```bash
npm install -g firebase-tools
firebase login
cd مجلد-مشروعك
firebase init functions      # اختر مشروعك، JavaScript، ولا تكتب فوق index.js إن سألك
```
ثم انسخ الملفين المرفقين إلى مجلد `functions/`:
- `functions/index.js`
- `functions/package.json`
ثم:
```bash
cd functions && npm install && cd ..
firebase deploy --only functions
```

## المرحلة ٥ — قواعد الأمان
في `firestore.rules` أُضيفت مجموعة `customer_tokens`. أعد لصق القواعد المحدّثة في Console → Firestore → Rules → Publish.

## المرحلة ٦ — الاختبار
1. على هاتف الزبون: ادخل بحسابه، واضغط لافتة **«فعّل الإشعارات»** واسمح بالإذن.
2. من حساب المالك: سجّل فاتورة جديدة لنفس الزبون (تأكد أن اسم الزبون مطابق لحسابه المفعّل حتى يُختم `customerId`).
3. أغلق تطبيق الزبون → يجب أن يصله إشعار خلال ثوانٍ.

## ملاحظات
- على iPhone: إشعارات الويب تعمل فقط إذا ثُبّت التطبيق على الشاشة الرئيسية (PWA) ومن iOS 16.4+.
- إن لم يصل الإشعار: تحقق من سجلّ الدالة (Console → Functions → Logs)، ومن وجود مستند في `customer_tokens`، ومن أن الفاتورة تحمل `customerId`.
- يُرسَل الإشعار فقط للزبائن **المسجّلين** (لهم حساب) والذين فعّلوا الإشعارات على أجهزتهم.
